# useState Hook Демонстрация

Този проект демонстрира използването на React Hook `useState` с различни примери и сценарии. Проектът е изграден с TypeScript и Vite.

## Какво е useState?

`useState` е React Hook, който позволява на функционалните компоненти да създават и управляват вътрешно състояние (state). Преди появата на хуковете в React 16.8, само класовите компоненти можеха да имат собствено състояние, но с хуковете това стана възможно и за функционалните компоненти.

## Синтаксис на useState

```typescript
const [state, setState] = useState(initialState);
```

- **state**: Това е текущата стойност на състоянието.
- **setState**: Това е функция, която променя стойността на състоянието. Когато я извикате, React актуализира състоянието и рендерира компонента отново.
- **initialState**: Началната стойност на състоянието, която може да бъде число, низ, обект, масив или друга стойност.

## Компоненти в проекта

### 1. Counter (Основен брояч)
Прост брояч, който демонстрира основното използване на useState:
- Декларира state променлива `count` с начална стойност 0
- При натискане на бутона увеличава стойността
- Показва как React автоматично рендерира компонента при промяна на състоянието

### 2. UserProfile (Потребителски профил)
Демонстрира използването на множество `useState` хукове в един компонент:
- Управлява име и възраст като отделни състояния
- Показва как различни части на състоянието могат да се променят независимо
- Включва текстово поле за промяна на името

### 3. AdvancedCounter (Усъвършенстван брояч)
Показва правилния начин за промяна на състоянието в зависимост от предишното:
- Използва функция `prevCount => prevCount + 1` вместо директна стойност
- Гарантира правилни изчисления при асинхронни операции
- Включва бутони за различни операции (+1, -1, +5, -5, нулиране)

## Защо да използваме useState?

**Лесно управление** на динамични стойности като броячи, формуляри, списъци и др.

**Функционални компоненти** с хукове са по-компактни и по-лесни за разбиране в сравнение с класовите компоненти.

**Модулност**: Всеки useState управлява конкретна част от състоянието, което прави кода по-лесен за поддръжка.

## Основни характеристики на useState

- При промяна на състоянието React автоматично рендерира само засегнатите части от интерфейса
- Може да се комбинира с други хукове като `useEffect`, за да се постигне по-сложна логика
- useState приема начална стойност. Ако тя изисква тежка операция, можете да използвате функция:
  ```typescript
  const [state, setState] = useState(() => сложнаИнициализация());
  ```

## Чести грешки при useState

Не правете това:
```typescript
state = newValue;
```
Вместо това използвайте `setState`.

React не обновява състоянието моментално. Промените се насрочват и React оптимизира процеса на рендериране.

Когато състоянието зависи от предишно, използвайте функция вместо директна стойност в setState:
```typescript
setCount((prevCount) => prevCount + 1) // Правилно
setCount(count + 1) // Може да причини проблеми при async операции
```
